1.2 Create compatibility_checker.py
Prompt:

Write compatibility_checker.py with deterministic compatibility checks:

check_cpu_motherboard(cpu, mobo): compares socket.

check_ram_motherboard(ram, mobo): compares DDR generation.

check_psu_wattage(build): ensures PSU wattage >= sum of TDPs + 150W.

check_gpu_case(gpu, case): compares GPU length to case max.

validate_build(build): returns {"passed": bool, "errors": list} with error codes and fix suggestions.
Use only Python, no LLM.

<details> <summary>✅ Code snippet (copy‑paste if needed)</summary>
python
def check_cpu_motherboard(cpu, mobo):
    return cpu["socket"] == mobo["socket"]

def check_ram_motherboard(ram, mobo):
    return ram["ddr_gen"] == mobo["ddr_support"]

def check_psu_wattage(build):
    total_tdp = (build["cpu"]["tdp"] + build["gpu"]["tdp"] +
                 build["mobo"].get("tdp", 50) + build["ram"].get("tdp", 5))
    return build["psu"]["wattage"] >= total_tdp + 150

def check_gpu_case(gpu, case):
    return gpu["length_mm"] <= case["max_gpu_length_mm"]

def validate_build(build):
    errors = []
    if not check_cpu_motherboard(build["cpu"], build["mobo"]):
        errors.append({"code": "SOCKET_MISMATCH",
                       "message": f"CPU socket {build['cpu']['socket']} incompatible with motherboard socket {build['mobo']['socket']}",
                       "fix": "Replace motherboard with compatible socket"})
    if not check_ram_motherboard(build["ram"], build["mobo"]):
        errors.append({"code": "RAM_GEN_MISMATCH",
                       "message": f"{build['ram']['ddr_gen']} RAM on {build['mobo']['ddr_support']} motherboard",
                       "fix": "Replace RAM with correct generation"})
    if not check_psu_wattage(build):
        errors.append({"code": "INSUFFICIENT_POWER",
                       "message": "PSU wattage too low for this build",
                       "fix": "Upgrade PSU by at least 150W"})
    if not check_gpu_case(build["gpu"], build["case"]):
        errors.append({"code": "GPU_CLEARANCE_FAIL",
                       "message": f"GPU length {build['gpu']['length_mm']}mm exceeds case max {build['case']['max_gpu_length_mm']}mm",
                       "fix": "Choose a larger case or shorter GPU"})
    return {"passed": len(errors) == 0, "errors": errors}

🤖 Phase 2: Three Agents (Hours 4‑8)
2.1 Create agents.py with CrewAI
Prompt:

Create a CrewAI system with three agents defined in agents.py. Use the following agent roles and prompts. Load parts.json. Use compatibility_checker for validation. Implement a main function that runs the crew with a user input string.

Agent 1 – Analysis Agent
Role: Parse user request into structured data.
Prompt: (exact as Claude gave)

text
You are a PC building analyst. Given a user's request, extract:
1. Total budget in USD
2. Primary use case (gaming / creative work / general productivity / development)
3. Performance priority (max fps / max quality / balanced)
4. Any stated preferences or constraints

Return ONLY a JSON object:
{
  "budget": <number>,
  "use_case": <string>,
  "priority": <string>,
  "constraints": [<string>]
}

User request: {user_input}
Agent 2 – Recommendation Agent
Role: Select parts based on analysis and any validation error.
Prompt:

text
You are a PC parts selector. You have access to parts.json.
Given this build analysis: {analysis}
And this validation error (if any): {error}

Select one part from each category that:
- Fits within the budget allocation
- Matches the use case tier
- Does NOT repeat any part that previously caused a validation error

Budget allocation rules:
- Gaming: 40% GPU, 20% CPU, 15% mobo, 10% RAM, 10% PSU, 5% case
- Creative: 25% GPU, 30% CPU, 15% mobo, 15% RAM, 10% PSU, 5% case
- General: 15% GPU, 35% CPU, 20% mobo, 15% RAM, 10% PSU, 5% case

Return ONLY a JSON object with selected part IDs.
Agent 3 – Validation Agent
Role: Check compatibility and either pass or return error.
Prompt:

text
You are a build validator. You receive a proposed parts list.
Run compatibility_checker.validate_build() on it.
If it passes, return: {"status": "PASSED", "build": <build>}
If it fails, return: {"status": "FAILED", "errors": <errors>, "instruction": "Replace <part> — reason: <reason>"}

You do NOT guess compatibility. You only report what the checker returns.
Maximum 3 retry loops before escalating to user.
2.2 Wire the Crew
In crew.py (or same file), set up sequential tasks:

Analysis Task → output to analysis_result

Recommendation Task → uses analysis_result and any error from previous loop; outputs parts list.

Validation Task → runs validate_build on parts. If fails, feed error back to Recommendation Task (max 3 retries). If passes, finalize.
